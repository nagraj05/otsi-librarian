## вңҰ What is OTSI Librarian?

OTSI Librarian is a self-service book lending system for the office. Users sign up, browse the catalog, and request books вҖ” the admin approves or rejects requests and hands the book over physically. Once a book is in hand, users log their daily reading to build streaks and climb the leaderboard.

Admins can also attach an EPUB file to any book in the catalog, making it instantly readable in the built-in ebook reader вҖ” with bookmarks, highlights, theme switching, and automatic resume. Users also have a private personal shelf to track books they own at home, with status, progress, half-star ratings, and dates.

Everything is in-app: no WhatsApp, no email вҖ” notifications appear in the bell icon and update automatically every 20 seconds via polling.

## вңҰ Database Schema

| Table | Purpose |
|---|---|
| `users` | Clerk-synced users вҖ” id, name, email, username, role, image_url |
| `borrows` | Borrow records вҖ” status (`pending/active/returned/rejected`), dates, book snapshot |
| `books` | Library catalog вҖ” metadata + `ebook_url` (Vercel Blob) |
| `reading_logs` | Daily pages-read per borrow вҖ” drives streaks and leaderboard |
| `notifications` | Per-user alerts for approvals, rejections, waitlist |
| `ebook_progress` | Last EPUB position (CFI) per user+book вҖ” one row, UPSERT |
| `ebook_bookmarks` | User bookmarks: CFI + chapter label |
| `ebook_highlights` | User highlights: CFI range + text + colour (yellow/green/blue/pink) |
| `personal_books` | Personal shelf вҖ” status, current page, dates, rating (NUMERIC 0.5вҖ“5), private notes |

Run `/api/migrate` once after first deploy (and after any update) to apply all schema changes idempotently.

## вңҰ How It Works

**Borrow flow**
```
User requests a book
  в””в”Җв–ә requestBook() server action
        в””в”Җв–ә INSERT INTO borrows (status = 'pending')
              в””в”Җв–ә notify admin via notifications table

Admin approves
  в””в”Җв–ә approveRequest(borrowId, takenDate, dueDate)
        в””в”Җв–ә UPDATE borrows SET status = 'active'
              в””в”Җв–ә notify user вҶ’ bell updates within 20s

User logs reading (by current page number)
  в””в”Җв–ә logReading(borrowId, pagesReadDelta)
        в””в”Җв–ә UPSERT reading_logs (one row per user/book/day)
              в””в”Җв–ә streak recalculated on next dashboard load

Book returned
  в””в”Җв–ә markReturned(borrowId)
        в””в”Җв–ә UPDATE borrows SET status = 'returned'
              в””в”Җв–ә notify next person on waitlist if any
```

**Ebook flow**
```
Admin uploads EPUB
  в””в”Җв–ә Vercel Blob client upload (drag-drop, progress bar)
        в””в”Җв–ә blob URL stored in books.ebook_url

User opens reader (/read/[bookId])
  в””в”Җв–ә Server loads book + saved CFI position + bookmarks + highlights
        в””в”Җв–ә ReaderClient renders epub.js full-screen
              в”ңв”Җв–ә Position auto-saved (debounced 1.5s) on every page turn
              в”ңв”Җв–ә Text select вҶ’ colour picker вҶ’ highlight saved + rendered
              в””в”Җв–ә Bookmark icon вҶ’ saves current CFI with chapter label
```

**Personal shelf flow**
```
User adds a book (search or manual)
  в””в”Җв–ә addPersonalBook() server action
        в””в”Җв–ә INSERT INTO personal_books

User updates progress
  в””в”Җв–ә updatePersonalBook(id, { status, current_page, ... })
        в””в”Җв–ә UPDATE personal_books SET ... updated_at = NOW()
```
