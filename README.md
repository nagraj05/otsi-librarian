## вңҰ What is OTSI Librarian?

OTSI Librarian is a self-service book lending system for the office. Users sign up, browse the catalog, and request books вҖ” the admin approves or rejects requests and hands the book over physically. Once a book is in hand, users log their daily reading to build streaks and climb the leaderboard.

Everything is in-app: no WhatsApp, no email вҖ” notifications appear in the bell icon and update automatically every 20 seconds via polling.

## вңҰ Getting Started

```bash
# 1. Clone
git clone https://github.com/nagraj05/otsi-librarian.git
cd otsi-librarian

# 2. Install dependencies
bun install

# 3. Configure environment
cp .env.example .env.local
# fill in the values below
```

**.env.local**

```env
NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY=pk_...
CLERK_SECRET_KEY=sk_...
DATABASE_URL=postgresql://...
ADMIN_CLERK_ID=user_...          # your Clerk user ID вҖ” grants admin role
GOOGLE_BOOKS_API_KEY=...         # optional
```

```bash
# 4. Set up the database (run once after starting the dev server)
#    Visit: http://localhost:3000/api/reset
#    (or /api/migrate to add columns to an existing DB)

# 5. Start
bun dev
```

Open [http://localhost:3000](http://localhost:3000) and sign in. If your Clerk ID matches `ADMIN_CLERK_ID` you'll have admin access automatically.

```bash
bun build      # production build
bun start      # serve production build
bun lint       # ESLint check
```

## вңҰ Routes

| Route | Access | Description |
|---|---|---|
| `/` | Public | Landing page вҖ” sign in to enter |
| `/dashboard` | Auth | Personal dashboard вҖ” active borrows, reading log, streak |
| `/catalog` | Auth | Browse all books вҖ” request, waitlist, availability status |
| `/leaderboard` | Auth | Ranked by reading streak + total pages |
| `/@username` | Auth | Public user profile вҖ” stats, currently reading, history |
| `/admin` | Admin only | Manage requests, active borrows, library catalog |
| `/api/books/search` | Internal | Google Books search proxy (`?q=query`) |
| `/api/notifications` | Internal | Returns current user's notifications (used by polling) |
| `/api/reset` | One-time | Drops and recreates all tables fresh |
| `/api/migrate` | One-time | Idempotent вҖ” adds new columns to existing tables |

## вңҰ How It Works

```
User requests a book
  в””в”Җв–ә requestBook() server action
        в””в”Җв–ә INSERT INTO borrows (status = 'pending')
              в””в”Җв–ә notify admin via notifications table

Admin approves
  в””в”Җв–ә approveRequest(borrowId, takenDate, dueDate)
        в””в”Җв–ә UPDATE borrows SET status = 'active'
              в””в”Җв–ә notify user вҶ’ bell updates within 20s

User logs reading
  в””в”Җв–ә logReading(borrowId, pagesRead)
        в””в”Җв–ә UPSERT reading_logs (one row per user/book/day)
              в””в”Җв–ә streak recalculated on next dashboard load

Book returned
  в””в”Җв–ә markReturned(borrowId)
        в””в”Җв–ә UPDATE borrows SET status = 'returned'
              в””в”Җв–ә notify next person on waitlist if any
```
